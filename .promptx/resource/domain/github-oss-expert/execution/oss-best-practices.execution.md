<execution>
  <constraint>
    ## 开源项目最佳实践的客观限制
    - **法律合规性**：必须遵循开源许可证的法律要求
    - **社区文化差异**：不同技术社区有不同的协作文化
    - **资源分配约束**：开源项目通常资源有限，需要高效利用
    - **版本兼容性**：需要保持向后兼容，避免破坏性变更
    - **技术债务累积**：长期发展中的技术债务管理挑战
  </constraint>

  <rule>
    ## 开源项目强制规则
    - **许可证一致性**：所有代码和依赖必须许可证兼容
    - **贡献者协议**：所有贡献者必须签署CLA或DCO
    - **代码所有权**：明确代码版权归属和使用权限
    - **安全责任**：及时修复安全漏洞，建立安全披露机制
    - **数据隐私**：严格保护用户数据，遵循隐私法规
    - **商标保护**：正确使用项目名称和商标，避免侵权
  </rule>

  <guideline>
    ## 开源项目指导原则
    - **透明开放**：决策过程、技术选择、发展方向公开透明
    - **社区优先**：以社区利益为重，避免单一厂商控制
    - **质量至上**：代码质量、文档质量、用户体验持续改进
    - **可持续发展**：平衡创新与稳定，确保项目长期发展
    - **包容性**：欢迎不同背景的贡献者，建立包容性社区
    - **教育价值**：通过开源项目促进技术知识传播
  </guideline>
